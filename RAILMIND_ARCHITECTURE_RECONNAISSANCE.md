## 6. Ultimate System Architecture

**Core Philosophy:** A distributed, event-driven, microservices architecture utilizing a central Kafka message bus, allowing loosely coupled agents, data streams, and UI components to interact in real-time.

**Layers:**
- **Ingestion & Edge:** Apache NiFi / Fluentd pulling from APIs, MQTT from IoT sensors.
- **Message Broker:** Apache Kafka (Handling streams: Telemetry, Logs, Agent Thoughts, UI Events).
- **Processing & Orchestration:** Temporal for deterministic workflow execution; LangGraph for stateful agent reasoning loops.
- **Data Stores:**
  - *Telemetry:* ClickHouse (Fast time-series analytics).
  - *Graph/Relational:* Neo4j + PostgreSQL (Asset topology and structured data).
  - *Vector/Search:* Qdrant / Typesense (Semantic embeddings and fast text search).
  - *Memory:* Redis (Fast state and agent working memory).
- **Inference & AI:** vLLM / Ollama for local fast inference; API gateways to Claude/OpenAI for heavy reasoning.
- **Backend API:** FastAPI (Async REST and WebSockets).
- **Frontend/UI:** React + Vite, Zustand (State), React Flow (Graph Visualization), WebGL (Maps).

## 7. Agent Architecture

**Inspiration:** CrewAI (Personas) + LangGraph (State/Control Flow) + MemGPT (Memory)

- **Supervisor Agent (The "Conductor"):** Monitors the overall state, delegates tasks to specialized agents, and resolves conflicts (e.g., Maintenance wants to close a track, Operations needs it open).
- **Telemetry Analyst Agent:** Continuously consumes ClickHouse streams, performing statistical anomaly detection. Triggers the Supervisor upon threshold breaches.
- **Investigator Agent (GraphRAG):** When an anomaly occurs, queries Neo4j and Qdrant to find historical precedents and affected assets.
- **Planner Agent (Rerouting):** Uses constraint programming and LLM heuristics to generate alternative routes.
- **Critic / Verification Agent:** A strictly rule-based/neuro-symbolic agent that checks the Planner's output against hard safety rules (e.g., speed limits, train length vs. siding length).
- **Communications Agent:** Drafts and dispatches targeted Twilio SMS, emails, and station announcements tailored to the audience (e.g., technical jargon for maintenance, simple instructions for passengers).
- **Memory Manager Agent:** Runs asynchronously to summarize completed incidents and update long-term vector/graph stores (Episodic and Semantic memory consolidation).

## 8. Data Architecture

**Inspiration:** Lambda Architecture + Bi-Temporal Modeling

- **Hot Path (Real-Time):** Telemetry -> Kafka -> In-Memory State (Redis) & Telemetry Analyst Agent -> WebSocket to UI. Latency < 100ms.
- **Warm Path (Interactive Analytics):** Kafka -> ClickHouse / DuckDB. Used by the UI for zooming into recent timelines and generating dashboards.
- **Cold Path (Historical & ML Training):** Batch processing to S3/Delta Lake. Used for training predictive models and deep historical analysis.
- **Knowledge Graph (Topology):** Nodes represent Stations, Switches, Signals, Trains. Edges represent physical connections, communication links, and current assignments.
- **Vector Embeddings:** Every incident report, maintenance manual, and agent decision rationale is embedded and stored for semantic retrieval.

## 9. UI Architecture

**Inspiration:** Palantir Foundry + Grafana + Superset

- **The Infinite Canvas:** The primary workspace is not a series of pages, but an infinite digital twin canvas (React Flow / WebGL). Operators zoom out to see the whole country, zoom in to see a specific switch.
- **Overlay Layers:** Data is toggled via layers (e.g., Weather overlay, Delay Heatmap overlay, Maintenance overlay).
- **The "Timeline Scrub":** A global timeline control at the bottom. Dragging it back in time changes the state of *all* widgets and the map to exactly what it was at that moment.
- **Agent Copilot Drawer:** A persistent side-panel showing the "Thought Process" of active agents (Langfuse style), allowing human intervention or approval.
- **Dynamic Workspaces:** Users can snap widgets together to create custom dashboards for specific crises and share the URL with colleagues for a synchronized view.
- **Command Palette:** `Ctrl+K` for instant access to any train, station, or action. No digging through menus.
